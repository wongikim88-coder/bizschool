## 1. Overview

### 1.1 Purpose

비즈스쿨 플랫폼의 비전, 핵심 기능, 차별점을 방문자에게 명확히 전달하는 소개 페이지를 제작한다. 메인 카피문구와 소개글을 통해 플랫폼의 정체성을 각인시키고, 신규 유저의 가입 전환을 유도한다.

### 1.2 Background

비즈스쿨은 세무사, 회계사 등 전문가가 강의를 업로드하고, 수강생이 업무 노하우를 보강할 수 있는 **오픈러닝 플랫폼**이다. 현재 Footer에 `/about` 링크가 존재하지만 실제 페이지가 없는 상태이며, 플랫폼의 컨셉을 전달할 소개 페이지가 필요하다.

**플랫폼 핵심 컨셉:**
- **강의**: 전문가가 자유롭게 강의를 업로드, 수강생이 선택 수강
- **전문가상담**: AI 기반으로 업무 궁금증을 전문가에게 질문/검증
- **커뮤니티**: 업무 노하우 공유, 댓글 소통
- **선순환 구조**: 콘텐츠 업로드 → 피드백 → 성장의 오픈러닝 생태계

### 1.3 Related Documents

- 소개페이지 작성지침: `소개페이지 작성지침.txt`
- 기존 메인 페이지: `src/app/page.tsx`
- Footer (about 링크 존재): `src/components/layout/Footer.tsx`

## 4. 카피문구 및 소개글

### 4.1 메인 카피문구 (Hero)

**메인 카피:**
> **일하면서 성장하는 가장 확실한 방법**

**서브 카피:**
> 세무, 회계, 경영 전문가의 실무 강의부터 AI 전문가상담, 커뮤니티까지.
> 비즈스쿨에서 당신의 업무 역량을 한 단계 끌어올리세요.

### 4.2 핵심 기능 소개글

**강의 (Courses)**
- 제목: "전문가의 실무 노하우를 내 것으로"
- 설명: 세무사, 회계사 등 현직 전문가가 직접 만든 강의를 자유롭게 수강하세요. 업무에 바로 적용할 수 있는 실전 중심 커리큘럼입니다.

**전문가상담 (Expert Consultation)**
- 제목: "궁금한 건 바로 물어보세요"
- 설명: 업무 중 막히는 부분, 판단이 어려운 사안을 AI 전문가상담으로 해결하세요. 전문가의 경험에 기반한 답변을 빠르게 받아볼 수 있습니다.

**커뮤니티 (Community)**
- 제목: "함께 나누면 더 빨리 성장합니다"
- 설명: 업무 노하우를 공유하고, 동료들의 경험에서 배우세요. 질문, 사례 공유, 자유 토론으로 함께 성장하는 공간입니다.

### 4.3 선순환 구조 소개글

- 제목: "배우고, 나누고, 함께 성장하는 선순환"
- 설명: 비즈스쿨은 단순한 강의 플랫폼이 아닙니다. 전문가가 지식을 나누고, 수강생이 배운 것을 실무에 적용하고, 그 경험을 다시 커뮤니티에 공유하는 — 모두가 함께 성장하는 오픈러닝 생태계입니다.

### 4.4 CTA 소개글

- 제목: "지금 바로 시작하세요"
- 설명: 비즈스쿨에서 당신의 성장 여정을 시작하세요.
- 버튼: "무료로 시작하기"

## 5. 페이지 구성 (Section Flow)

```
┌─────────────────────────────────────────────┐
│  1. Hero Section                            │
│     - 메인 카피 + 서브 카피                    │
│     - 배경: 그라데이션 또는 일러스트             │
│     - CTA 버튼: "시작하기"                     │
├─────────────────────────────────────────────┤
│  2. Features Section                        │
│     - 3-column 카드 (강의/전문가상담/커뮤니티)   │
│     - 아이콘 + 제목 + 설명                     │
├─────────────────────────────────────────────┤
│  3. Cycle Section (선순환 구조)               │
│     - 순환 다이어그램 또는 스텝 표현             │
│     - 업로드 → 피드백 → 성장                   │
├─────────────────────────────────────────────┤
│  4. CTA Section                             │
│     - 최종 행동 유도                           │
│     - "무료로 시작하기" 버튼                    │
└─────────────────────────────────────────────┘
```

## 6. Success Criteria

- [ ] `/about` 라우트에서 소개 페이지가 정상 렌더링

- [ ] Tailwind CSS v4 규칙 준수 (`var()` 래퍼 사용)
- [ ] 시맨틱 HTML 구조

## 7. Risks and Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| 카피문구가 플랫폼 컨셉을 충분히 전달하지 못함 | Medium | Medium | 작성 후 사용자 피드백 반영 |
| 기존 페이지와 디자인 톤앤매너 불일치 | Medium | Low | 동일 CSS 변수, 컴포넌트 패턴 재사용 |
| 선순환 구조 시각화가 복잡해짐 | Low | Medium | 심플한 아이콘 + 화살표 방식 채택 |

## 8. Architecture Considerations

### 8.1 Project Level Selection

| Level | Characteristics | Selected |
|-------|-----------------|:--------:|
| **Starter** | 정적 소개 페이지, 백엔드 불필요 | **O** |

### 8.2 Key Architectural Decisions

| Decision | Selected | Rationale |
|----------|----------|-----------|
| 라우팅 | Next.js App Router (`/about`) | 기존 프로젝트 구조와 동일 |
| 스타일링 | Tailwind CSS v4 + CSS 변수 | 기존 디자인 시스템 활용 |
| 아이콘 | Lucide React | 기존 프로젝트에서 사용 중 |
| 컴포넌트 구조 | 단일 page.tsx + 인라인 섹션 | 소개 페이지 특성상 단일 파일이 적합 |

### 8.3 파일 구조

```
src/app/about/
  page.tsx          ← 소개 페이지 (메인 파일)
```
