# Plan: 콘텐츠 업로드 (콘텐츠 유형 선택)

> **Feature**: content-upload
> **Created**: 2026-03-27
> **Status**: Plan

## 1. 개요

전문가(expert) 계정으로 로그인 시 헤더의 "콘텐츠 업로드" 버튼을 클릭하면 `/expert/upload` 페이지로 이동한다.
첫 페이지에서는 업로드할 콘텐츠 유형을 선택하는 카드 UI를 제공한다.

## 2. 현재 상태

- 헤더에 `role === "expert"` 조건으로 "콘텐츠 업로드" 버튼이 이미 존재 (`/expert/upload` 링크)
- `/expert/upload` 라우트 페이지는 아직 미구현 (404)

## 3. 요구사항

### 3.1 접근 조건
- `session.user.role === "expert"` 인 경우에만 접근 가능
- 비로그인 또는 일반 사용자는 접근 시 리다이렉트 (로그인 페이지 또는 메인)

### 3.2 콘텐츠 유형 선택 페이지 (`/expert/upload`)

참조 이미지 기반 UI 구성:

1. **인사말 영역**
   - "반갑습니다. {사용자이름} 님!"
   - "어떤 강의를 만들어볼까요?" 부제

2. **콘텐츠 유형 카드** (2개만 구현)
   - **강의**: 아이콘 + 제목 + 설명 + 해시태그 + 라디오 선택
     - 설명: "원하는 시간에 자유롭게 학습하는 온라인 강의"
     - 부가설명: "영상, 글, 미션, 자동 메세지 등을 통해 효과적인 학습 커리큘럼을 만듭니다."
     - 태그: #상시 수업 #영상 수업
   - **컬럼**: 아이콘 + 제목 + 설명 + 해시태그 + 라디오 선택
     - 설명: "작지만 강력한 지식 한 조각"
     - 부가설명: "클립은 글, 이미지, PDF, 템플릿 등 다양한 형태의 콘텐츠를 쉽게 만들고, 필요한 사람에게 가치를 전달할 수 있어요."
     - 태그: #템플릿 #에셋 #아티클

3. **레이아웃**
   - 2열 그리드 (데스크톱), 1열 (모바일)
   - 카드 선택 시 테두리 색상 변경 + 체크 표시
   - 기본 선택: "강의"

4. **하단 영역**
   - 스텝 인디케이터 (5개 도트 중 1번째 활성)
   - "다음" 버튼 (현재는 동작 없이 UI만 구현)

## 4. 영향 범위

| 파일 | 변경 내용 |
|------|----------|
| `src/app/expert/upload/page.tsx` | **신규** - 콘텐츠 유형 선택 페이지 |
| `src/components/expert/ContentTypeCard.tsx` | **신규** - 콘텐츠 유형 카드 컴포넌트 |

## 5. 구현 순서

1. `/expert/upload/page.tsx` 페이지 생성 (접근 권한 체크 포함)
2. `ContentTypeCard` 컴포넌트 생성 (선택 가능한 카드 UI)
3. 콘텐츠 유형 선택 + 스텝 인디케이터 + 다음 버튼 조합

## 6. 기술 사항

- **권한 체크**: `useSession` 기반 클라이언트 사이드 체크, expert가 아닌 경우 리다이렉트
- **상태 관리**: `useState`로 선택된 콘텐츠 유형 관리
- **반응형**: `grid-cols-1 md:grid-cols-2`
- **스텝 인디케이터**: 5단계 도트 (향후 업로드 플로우 확장 대비, 현재 1단계만 활성)
- **"다음" 버튼**: UI만 렌더링 (클릭 시 미구현 안내 또는 no-op)
